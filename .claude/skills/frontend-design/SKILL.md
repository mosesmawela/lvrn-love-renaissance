# Frontend Design Skill

## Overview

The **frontend-design** skill creates distinctive, production-grade frontend interfaces that avoid generic aesthetics.

## Design Thinking Process

Before coding, commit to a "BOLD aesthetic direction":

1. **Purpose**: Define the problem and user
2. **Tone**: Select from extremes like:
   - Brutally minimal
   - Maximalist chaos
   - Retro-futuristic
   - Organic/natural
3. **Differentiation**: Determine "What makes this UNFORGETTABLE?"
4. **Constraints**: Address technical requirements

## Implementation Guidelines

Produce working code (HTML/CSS/JS, React, Vue) that is "visually striking and memorable" with "meticulously refined" details:

### Typography
- Avoid generic fonts like Arial and Inter
- Opt for distinctive, unexpected, characterful font choices
- Use typography as a design element, not just text

### Color
- Dominant colors with sharp accents outperform timid, evenly-distributed palettes
- Create memorable color moments

### Motion
- Prioritize CSS-only solutions for HTML when possible
- Use scroll-triggering and hover states that surprise
- Add depth through thoughtful animations

### Spatial Composition
- Unexpected layouts
- Asymmetry
- Overlap
- Diagonal flow
- Break the grid intentionally

### Backgrounds
- Apply gradient meshes
- Noise textures
- Geometric patterns
- Layered transparencies

## Critical Constraints

NEVER use "generic AI-generated aesthetics" including:
- Overused font families (Inter, Roboto, etc.)
- Clichéd color schemes (purple gradients on white backgrounds)
- Generic stock layouts
- Cookie-cutter component designs

Match complexity to vision:
- Maximalist designs need elaborate code
- Minimalist designs need restraint and precision

## Key Principle

"Claude is capable of extraordinary creative work"—execute distinctive visions with precision rather than converging on common choices.
